## 🔄 Regras para Adaptar Arquivos

Quando um novo integrante confirmar que vai apresentar:

1. **Criar arquivo** `<nome-slug>.md` nesta pasta
2. **Copiar a estrutura** de um dos arquivos existentes (deivison ou rios)
3. **Preencher os slides** que a pessoa fala com detalhe
4. **Incluir contexto** do que os outros falam (em itálico/bloco de citação)
5. **Adicionar lacunas** para o conteúdo dos slides que ainda não foi uploadado
6. **Atualizar esta tabela** de integrantes
7. **Commit + push** com mensagem: `feat: prompt individual <nome> para apresentação MinhaFila`

```markdown
[Tabela resumo de TODOS os 15 slides, destacando os seus]

[Texto completo da fala]

## ✅ Checklist Pessoal

- [ ] Li meu roteiro completo
- [ ] seiexatamente quando entro e saio
- [ ] Testei o fluxo da apresentação com os slides
- [ ] Pratiquei minha fala em voz alta (cronometrado)
```

---

## 🎨 Notas de Formatação

- **Fala do próprio usuário:** texto normal, sem formatação especial
- **Contexto do que outros falam:** usar bloco de citação (`>`) + itálico
- **Lacunas para preencher:** usar `[LACUNA — descrição]` em negrito
- **Dicas de apresentação:** usar badge `💡` antes do texto
- **Alertas/atenção:** usar badge `⚠️` antes do texto
